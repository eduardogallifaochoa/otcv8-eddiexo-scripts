## What’s in here

- `vBot/`, `cavebot/`, `targetbot/`: base bot modules/scripts
- `_Loader.lua`: profile entrypoint (loads UI styles + scripts)
- `*_configs/` and `storage/`: configs (server/profile dependent)

## Where does my Lua code go?

Usually:
- put your module in a proper folder (`scripts/`, `bot_loaders/`, `ui/`, etc)
- wire it from `_Loader.lua` (directly or via a loader)

## Fast install on another bot/profile

Example target profile:
`C:\Users\USER\AppData\Roaming\OTClientV8\Baiak Eternium\bot\DRUID`

### Option A (no code edits, recommended)
1. Go to **GitHub > Actions > Build Toolkit Artifacts**.
2. Download artifact `druid-toolkit-files`.
3. Extract and copy these files into target profile folder:
- `_Loader.lua`
- `druid_toolkit_single.lua`
4. Start OTCv8 with that profile. It autoloads.

### Option B (if you do not want to replace `_Loader.lua`)
- Copy only `druid_toolkit_single.lua`.
- Load manually once from Ingame Script Editor:
```lua
dofile('druid_toolkit_single.lua')
```

## GitHub artifact (loader/single-file)

Workflow added:
- `.github/workflows/build-artifacts.yml`

How to use:
1. Open **Actions** in GitHub.
2. Run **Build Toolkit Artifacts** (or use the run generated by push).
3. Download artifact `druid-toolkit-files`.
4. Zip contains:
- `_Loader.lua` (autoload ready)
- `druid_toolkit_single.lua` 
- `QUICKSTART.txt`

## Should we delete everything and keep only the loader?

Not recommended.

Keep full source repo (scripts/UI/builder), and distribute only artifact/single-file.
That gives you clean maintenance plus easy portable installs.
